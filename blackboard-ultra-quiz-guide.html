◯ How to Add a Multiple Choice Question
1
Click Add Question → Multiple Choice
Inside the test editor, click the + Add Question button and choose Multiple Choice.
2
Type Your Question Text
Click in the Question Text field and type your question. You can format text, add code, images, or math formulas using the toolbar.
Question 1 — Multiple Choice
Question Text *
What does dynamic binding in Java use to decide which method version to call at runtime?
Answer Options:
The reference type of the variable
The actual type of the object at runtime ✅
The method name only
The number of parameters
Points:
1
Save Question
3
Add Answer Options & Mark the Correct Answer
Type each answer option. Click the circle/checkbox next to the correct answer to mark it as correct — it will turn green with a checkmark ✅. Set the point value (default is 1).
4
Click Save Question
Click Save Question to save it. Repeat steps 1–4 for every question you need.
⚠ How to Add a True / False Question
1
Click Add Question → True/False
Type the statement. Then click True or False to indicate which is correct. Set point value and save.

Statement *
Dynamic binding uses the reference type to decide which method to call.

False ✅
⚠️
Important — Shuffle Answers: For Multiple Choice questions, enable Shuffle Answers in the question settings so the answer order is randomized for each student. This discourages answer sharing.

Configure Quiz Settings
Set the due date, time limit, attempts, and display options
1
Click the Settings (gear icon ⚙) on the Test
From the test editor page, click the gear icon ⚙ or the … (three dots) menu in the top-right corner, then choose Settings.
2
Configure the Key Options
A settings panel slides open on the right. Here are the most important options:
Test Settings
📅 Due Date Apr 20, 2026   11:59 PM
⏰ Time Limit 30 minutes
🔄 Number of Attempts 1 attempt
🔐 Shuffle Questions
👁 Show Correct Answers
📊 Show Score After Submit
🔒 Require a Password
3
Save Settings & Publish the Quiz
When you are satisfied with the settings, click Save. Then use the toggle at the top of the test to switch from Hidden to ✅ Visible to Students when you are ready for students to see it.
💡
Keep it Hidden while building: Always keep the quiz Hidden until you have finished adding all questions and reviewing everything. Students cannot see hidden assessments.

Export Questions as a QTI 2.1 Package
Save your question bank so you can reuse it in another course or semester
📌
How this works: In Blackboard Ultra, questions are shared via Question Banks — not via course-level export. You first copy your quiz questions into a Question Bank, then export the bank as a QTI 2.1 ZIP package.
1
In the right pane, under "Question Banks", click Manage banks
On the Course Content page, look at the right side panel. You will see a Question Banks section with a "Manage banks" link — click it.

Import QTI Package & Create the Test
Import the question bank, then build a new test from it
📌
How this works: You import the QTI ZIP into Question Banks (right pane), then create a new Test in Course Content and pull questions from the bank using Reuse Questions.
1
In the right pane, click Question Banks → Manage banks
On the Course Content page, find the Question Banks section in the right panel and click Manage banks.
2
Click the purple line, then choose Import from QTI 2.1
Click the purple + line on the Question Banks page. From the menu, choose Import from QTI 2.1 package.

All questions are now added. Use the right pane to set:

9
Set the release condition (availability window)
Set when the test becomes available to students and when it closes. This controls the exact time window students can access the test.
⏰
Example: Release from 7:00 pm to 10:59 pm on the exam date — students outside this window cannot open the test.
